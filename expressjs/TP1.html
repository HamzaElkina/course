🟢 TP Express JS : Création d'une API REST

Ce travail pratique vous guidera dans la création d'une API REST avec Express.js. Vous apprendrez à configurer un serveur, créer des endpoints et tester votre API avec Postman.

🎯 Objectifs de ce TP :
  • Initialiser un projet Node.js
  • Installer et configurer Express.js
  • Créer un serveur HTTP avec Express
  • Comprendre les routes GET et POST
  • Développer une API de calcul (somme de deux nombres)
  • Tester les endpoints avec Postman
  • Comprendre les codes de statut HTTP
  • Documenter son travail avec des captures d'écran

📦 ÉTAPE 1 : Initialisation du projet

1.1 - Créer et initialiser le projet Node.js

💻 Créer un dossier "express-api" et initialiser un projet Node.js
📌 Indications :
  • Créez un dossier nommé "express-api"
  • Ouvrez un terminal dans ce dossier
  • Utilisez npm init -y pour initialiser le projet
  • Le fichier package.json est créé. À votre avis, quel est son rôle ?
💡 Aide : Le fichier package.json contient les métadonnées du projet et la liste des dépendances. Le flag -y accepte toutes les valeurs par défaut.

📦 ÉTAPE 2 : Installation d'Express.js

2.1 - Installer Express

📦 Installer le framework Express.js
📌 Indications :
  • Exécutez npm install express
  • Observez l'apparition du dossier node_modules et du fichier package-lock.json
💡 Aide : Express est un framework minimaliste pour Node.js qui facilite la création d'API et d'applications web. Que contient le dossier node_modules ?

2.2 - Installer nodemon (optionnel mais recommandé)

🔄 Installer nodemon pour le développement
📌 Indications :
  • Exécutez npm install --save-dev nodemon
  • Nodemon redémarre automatiquement le serveur à chaque modification
💡 Aide : --save-dev indique que nodemon n'est utile qu'en développement, pas en production.

📸 Capture d'écran n°2

Capturez le terminal montrant l'installation d'Express et de nodemon.

2.3 - Configurer les scripts dans package.json

📝 Modifier le fichier package.json pour ajouter des scripts de démarrage
📌 Indications :
  • Dans la section "scripts" du package.json, ajoutez :
  • "start": "node server.js" (pour la production)
  • "dev": "nodemon server.js" (pour le développement)
💡 Aide : Les scripts permettent de lancer votre application facilement avec npm run dev ou npm start.

🚀 ÉTAPE 3 : Création du serveur Express

3.1 - Créer le fichier server.js

📝 Créer un fichier server.js contenant un serveur Express de base
📌 Indications :
  • Importez express avec require('express')
  • Créez une instance de l'application
  • Définissez un port (ex: 3000)
  • Créez une route GET à la racine "/" qui renvoie "Hello World"
  • Démarrez le serveur avec app.listen()
💡 Aide : La méthode app.get() définit une route pour les requêtes HTTP GET. Le premier paramètre est l'URL, le second une fonction avec req (requête) et res (réponse). Utilisez res.send() pour renvoyer une réponse.

🚀 Lancer le serveur et vérifier son fonctionnement

💡 Aide : Si vous voyez "Hello World", votre serveur fonctionne ! Pour arrêter le serveur : Ctrl + C.

➕ ÉTAPE 4 : API de calcul de somme (GET)

4.1 - Créer l'endpoint GET /somme

📝 Créer une route GET qui calcule la somme de deux nombres
📌 Spécifications :
  • URL : /somme
  • Paramètres : a et b (transmis en query string)
  • Exemple : /somme?a=5&b=3
  • Réponse JSON contenant a, b et le résultat
  • Gérer les cas d'erreur (paramètres manquants ou non numériques)
💡 Aides :
  • Les paramètres query sont accessibles via req.query
  • Utilisez parseFloat() pour convertir en nombre
  • isNaN() permet de vérifier si une valeur est un nombre valide
  • En cas d'erreur, utilisez res.status(400).json({ error: "message" })
  • Pour une réponse réussie : res.json({ a, b, somme })

📤 ÉTAPE 6 : API de somme en méthode POST

6.1 - Ajouter le middleware express.json()

⚙️ Configurer Express pour lire le JSON du body
📌 Indications :
  • Ajoutez app.use(express.json()) avant vos routes
  • Ce middleware permet de parser automatiquement le JSON reçu
💡 Aide : Sans ce middleware, req.body serait undefined.

6.2 - Créer la route POST /somme

📝 Créer une route POST qui calcule la somme à partir du body
📌 Spécifications :
  • Méthode : POST
  • URL : /somme
  • Body (JSON) : {"a": 5, "b": 3}
  • Réponse : identique à la version GET
  • Gestion d'erreur similaire
💡 Aide :
  • Les données du body sont accessibles via req.body
  • Utilisez app.post() au lieu de app.get()
  • Pensez à vérifier l'existence des champs : if (a === undefined || b === undefined)

6.3 - Tester la route POST avec Postman

🧪 Tester l'API POST /somme avec Postman
📌 Indications pour le test POST :
  • Méthode : POST
  • URL : http://localhost:3000/somme
  • Onglet "Body" → sélectionner "raw" → format "JSON"

💡 Cas à tester :
  • Body valide avec deux nombres
  • Body avec un champ manquant
  • Body avec des chaînes de caractères
  • Body vide

⚠️ Dépannage rapide :
  • Port déjà utilisé → Changez le port (3001, 4000, etc.) dans server.js
  • req.body est undefined → Avez-vous ajouté app.use(express.json()) ?
  • Postman n'arrive pas à se connecter → Vérifiez que le serveur tourne (npm run dev)
  • Erreur 404 → Vérifiez l'URL et la méthode HTTP utilisée
