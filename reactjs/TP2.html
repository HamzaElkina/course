⚛️ TP React : Navigation avec React Router et API Bitcoin

Ce travail pratique vous guidera à travers la création d'une application React multi-pages utilisant React Router DOM pour la navigation et Axios pour l'appel d'une API externe. Vous apprendrez à structurer une application React complète.

🎯 Objectifs de ce TP :
  • Installer et configurer React Router DOM
  • Créer trois pages distinctes : Home, About et Bitcoin
  • Mettre en place la navigation entre les pages avec useNavigate et Link
  • Installer et utiliser Axios pour les appels API
  • Tester une API avec Postman avant de l'intégrer
  • Appeler l'API de mempool.space pour obtenir le prix du Bitcoin
  • Afficher dynamiquement les données reçues (EUR et USD)
  • Documenter son travail avec des captures d'écran

📦 PARTIE 1 : Installation des dépendances

1.1 - Installation de React Router DOM

🧭 Installer la bibliothèque de navigation pour React

React Router DOM est la bibliothèque standard pour gérer la navigation entre différentes pages dans une application React. Elle permet de créer une expérience utilisateur fluide sans rechargement de la page.

📌 Commande à exécuter :
$ npm install react-router-dom

1.2 - Installation d'Axios

📡 Installer la bibliothèque pour les appels API

Axios est un client HTTP basé sur les Promesses qui facilite les requêtes vers des API externes. Il est plus simple à utiliser que fetch() et offre des fonctionnalités supplémentaires.

$ npm install axios

🔧 PARTIE 2 : Configuration du routeur

2.1 - Modification de main.jsx

📄 Envelopper l'application avec BrowserRouter

Pour utiliser React Router, toute l'application doit être enveloppée dans le composant <BrowserRouter>. C'est lui qui permet de synchroniser l'interface avec l'URL du navigateur.

📌 À faire : Ouvrez le fichier src/main.jsx et modifiez-le pour ajouter BrowserRouter.
// Avant modification : ReactDOM.createRoot(document.getElementById('root')).render( <React.StrictMode> <App /> </React.StrictMode> ) // Après modification : import { BrowserRouter } from 'react-router-dom' ReactDOM.createRoot(document.getElementById('root')).render( <React.StrictMode> <BrowserRouter> <App /> </BrowserRouter> </React.StrictMode> )

🛣️ PARTIE 4 : Configuration des routes dans App.jsx

4.1 - Définir les routes

🔀 Importer et configurer Routes et Route

Dans App.jsx, vous allez importer les trois pages et définir les routes correspondantes.

📌 Éléments à importer depuis react-router-dom :
  • Routes - conteneur pour toutes les routes
  • Route - définit une association entre un chemin URL et un composant
import { Routes, Route } from 'react-router-dom' import Home from './pages/Home' import About from './pages/About' import Bitcoin from './pages/Bitcoin' function App() { return ( <div> <Routes> <Route path="/" element={<Home />} /> <Route path="/about" element={<About />} /> <Route path="/bitcoin" element={<Bitcoin />} /> </Routes> </div> ) } export default App

🧭 PARTIE 5 : Création du menu de navigation

5.1 - Ajouter un menu avec Link ou useNavigate

🔗 Créer une barre de navigation pour changer de page
📌 Deux approches possibles :
  • Link : composant qui crée un lien cliquable (recommandé pour les menus)
  • useNavigate : hook qui permet la navigation programmatique (utile après une action)

💡 Question : Quelle approche est la plus adaptée pour un menu de navigation simple ?
// Exemple avec Link : import { Link } from 'react-router-dom' <nav> <Link to="/">Accueil</Link> <Link to="/about">À propos</Link> <Link to="/bitcoin">Bitcoin</Link> </nav> // Exemple avec useNavigate : import { useNavigate } from 'react-router-dom' const navigate = useNavigate() <button onClick={() => navigate('/')}>Accueil</button>

💰 PARTIE 6 : Appel API dans la page Bitcoin

6.0 - Découverte de l'API avec Postman

🔧 Tester l'API avant de l'intégrer dans React

Avant d'écrire du code, il est essentiel de comprendre comment fonctionne l'API que vous allez utiliser. Postman est un outil qui permet de tester des API facilement sans écrire une seule ligne de code.

📌 Qu'est-ce que Postman ?

Postman est une application qui permet de :

  • Envoyer des requêtes HTTP (GET, POST, PUT, DELETE...)
  • Visualiser les réponses des API
  • Tester les paramètres et les en-têtes
  • Debugger et comprendre la structure des données

🖥️ Étape 1 : Installation de Postman

Rendez-vous sur https://www.postman.com/downloads/ et téléchargez la version correspondant à votre système d'exploitation. Lancez l'installation et ouvrez Postman.

📡 Étape 2 : Création d'une nouvelle requête

Dans Postman, cliquez sur le bouton "New" puis "HTTP Request".

Configurez la requête comme suit :

  • Méthode HTTP : GET (défiler ou cliquer sur le dropdown)
  • URL : https://mempool.space/api/v1/prices

🚀 Étape 3 : Envoi de la requête

Cliquez sur le bouton bleu "Send".

Observez la réponse qui apparaît dans la partie inférieure.

📡 Réponse normale de l'API :

  "time": 1776715504,
  "USD": 76274,
  "EUR": 64744,
  "GBP": 56376,
  "CAD": 104092,
  "CHF": 59386,
  "AUD": 106182,
  "JPY": 12129540
}

🔍 Étape 4 : Analyse de la réponse

Observez attentivement la structure de la réponse :

  • La réponse est un objet JSON
  • Il contient la clé time (timestamp Unix)
  • Il contient des clés pour chaque devise : USD, EUR, GBP, etc.
  • Les valeurs sont des nombres (prix du Bitcoin dans la devise correspondante)

⚠️ Dépannage rapide :
  • Les routes ne fonctionnent pas → Vérifiez que BrowserRouter est bien présent dans main.jsx
  • Les liens ne changent pas la page → Vérifiez l'utilisation de "to" et non "href"
  • L'API ne répond pas → Vérifiez votre connexion internet et l'URL de l'API
  • Les prix ne s'affichent pas → Vérifiez le nom des propriétés dans la réponse (USD, EUR)
  • Erreur CORS → L'API mempool.space autorise les requêtes cross-origin, pas de souci normalement
  • Postman affiche une erreur → Vérifiez l'URL et votre connexion internet
